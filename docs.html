Getting Started

CDevKit is a desktop toolkit for developers working with game engines, such as Roblox, Unity or Unreal Engine, Discord bots, local web servers, and more — all from a single unified interface. It runs entirely on your machine. There's no cloud account, no subscription, and no telemetry. Just download the installer and run it.

Installation

  • Download the installer

    Click the Download CDevKit button above to download the installer.

  • Run the installer

    Open the downloaded installer and follow the on-screen steps. It will automatically set up CDevKit and place all files in the right folders for you.

  • Launch CDevKit

    Once installed, open CDevKit using the desktop shortcut. You can also run CDevKit.exe directly from the install directory if preferred.

    Head to the Apps page to enable the integrations you want to use. Have fun!

Roblox — Api's

The Roblox integration gives you direct access to Open Cloud APIs — DataStore, Messaging, Places, and more — without leaving your dev environment.

Codespace

Codespace lets you write and manage your Roblox ModuleScripts from within CDevKit. Choose a sync folder, create and edit .luau files, and keep your code in sync with Roblox Studio using the built-in Script Sync feature.

How it works

You pick a local folder where your .luau files live. CDevKit reads all the files in that folder and shows them as tabs in the editor. When you create a new file, it's saved as a ModuleScript (.luau) in that folder. Edit the code and hit Save — the file is written to disk, and Roblox Studio's Script Sync picks up the change automatically.

Sync Folder

The local directory CDevKit reads and writes scripts to. Point this at the same folder Studio Script Sync is watching so changes flow between the app and Studio without any manual copying. Click Browse to pick a folder, or Open to reveal it in your file manager.

Scripts

All .luau files in the sync folder appear as tabs. Click a tab to open it, edit the code, then press Save. Use + New to create a new ModuleScript, or Delete to remove the currently open file.

Setup

  • Enable Script Sync in Roblox Studio

    In Studio, go to File → Beta Features and enable Script Sync. This is Roblox's built-in feature for syncing scripts between Studio and your local filesystem — no third-party plugins required. Full setup guide: create.roblox.com/docs/scripting/sync

  • Link a folder in Studio

    In the Studio Explorer, right-click the folder you want to sync and select Script Sync → Sync with Directory. Choose a local directory on your computer and click Save. Any .luau files saved to that folder will be reflected as ModuleScripts inside Studio.

  • Point CDevKit at the same folder

    On the Codespace page, click Browse and select the same folder. CDevKit will immediately list all .luau files found there as editable tabs.

  • Create and edit scripts

    Hit + New to create a ModuleScript, give it a name, and start writing. Press Save when done — Studio will pick up the change automatically.

Writing Luau — Roblox scripting basics

All files in Codespace are ModuleScripts. A ModuleScript must return a value — usually a table of functions or data — so other scripts can require() it. Read the full ModuleScript documentation →

ModuleScript structure

-- MyModule.luau
local MyModule = {}

function MyModule.greet(name: string): string
    return "Hello, " .. name .. "!"
end

return MyModule

Requiring a ModuleScript

From a Script or LocalScript inside Studio, use require() with the instance path to load your module:

local MyModule = require(game.ServerScriptService.MyModule)
print(MyModule.greet("World"))  -- Hello, World!
ModuleScripts are shared code — they don't run on their own. They need to be require()d by a Script (server) or LocalScript (client) inside Studio to execute. Always wrap DataStore calls in pcall to handle errors gracefully.

Console

The Console page captures all stdout and stderr output from the CDevKit process and displays it in a timestamped, colour-coded log viewer.

Controls

The Clear button wipes the log display and resets the entry counter. A system message is appended after clearing to confirm the action.

How it works

On startup, sys.stdout and sys.stderr are replaced with ConsoleStream wrappers that emit a Qt signal for each line. The signal is connected to the log widget's append method, making it thread-safe to print from background threads (e.g. the Flask server thread).

# Any print() call anywhere in the app appears here
print("Hello from CDevKit")       # → info level
print("Something broke", file=sys.stderr)  # → error level
